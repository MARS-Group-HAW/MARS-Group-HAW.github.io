MARS Documentation | Website

The whole documentation shall be generated after every update and is available at http://mars.haw-hamburg.de/

Prerequisites

For code documentation, we use the metadata tool the documentation tool docfx. Make sure you have installed the followin prerequisites on your local development machine

  • docfx - For MacOS it is important that you have install only the version 2.51, since newer version seems to have problems with netstandard2.0 project like MARS. The the required version can be downloaded from here is available here . All other please follow the Getting started page.

For making changes on the website.

  • docfx to create the documentation web-content from markdown and C# API.
  • Docker platform to create a local and global deployment.
  • kubectl to serve the local deployment and to apply change for the cluster.

Build via Script

The existing build.sh script contains all required commands to build a local Docker image of the current documentation and upload it to the remote Docker Hub.

By default each image is tagged with a successing version number. Apply this version to the build.sh call. Therefore change into the working directory of Documentation (when not already changed).

cd Documentation

./build.sh <version-number>

The version number can be queried by asking the current deployment at the cluster or looking at the latest version in the repo git.haw-hamburg.de:5005/mars/life/

When the build.sh script cannot be exeucted directly apply execution rights before chmod +x ./build.sh

Build with DocfX

Use the pre-defined configuration docfx.json and run:

docfx docfx.json --serve

When docfx is an exe file and your local machine is linux or macOS, please use mono

mono docfx.exe docfx.json --serve

This will generate the website and starts a local webserver for to show your results.

Custom Tutorials or Articles

To write your own tutorials and other useful articles, how to use MARS. Create a new markdown file and your write your contente therein. Move your article into the folder articles folder and link it by using the toc.yml (table of content) or by inserting the link into the into.md page at articles/intro.md.
